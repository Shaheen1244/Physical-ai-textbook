# Data Model: RAG Qdrant Integration

## BookContentChunk

Represents a section of book content that has been processed and embedded.

**Fields:**
- `id` (string): Unique identifier for the chunk
- `content` (string): The actual text content of the chunk
- `sourceFile` (string): Path to the original markdown file
- `sectionTitle` (string): Title of the section/chapter
- `position` (number): Position of the chunk within the document
- `embedding` (number[]): Vector embedding of the content
- `metadata` (object): Additional metadata like word count, headings hierarchy

**Relationships:**
- Part of a single book document
- Multiple chunks can belong to the same source file

## QdrantCollection

A vector database collection named "book" that stores content chunks with their embeddings for similarity search.

**Fields:**
- `collectionName` (string): "book" - the name of the collection
- `vectorsConfig`: Configuration for vector storage
- `points`: Individual vector points (BookContentChunk instances with embeddings)

## UserQuery

A question submitted by a user that needs to be answered using book content.

**Fields:**
- `id` (string): Unique identifier for the query
- `question` (string): The actual question text
- `timestamp` (date): When the query was made
- `userId` (string): Identifier of the user who made the query (optional)

## RAGResponse

An answer generated by the system based on retrieved book content.

**Fields:**
- `id` (string): Unique identifier for the response
- `queryId` (string): Reference to the original query
- `answer` (string): The generated answer based on book content
- `sourceChunks` (array): References to the book chunks used to generate the answer
- `confidenceScore` (number): Confidence level of the response
- `timestamp` (date): When the response was generated
